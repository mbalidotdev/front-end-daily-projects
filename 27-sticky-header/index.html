Section 2: How Sticky Works

The position: sticky property toggles between relative and fixed depending on the scroll position.

When the user scrolls past the element, it "sticks" to the top (or bottom, or left, or right) based on the top, bottom, left, or right value you set.

/* Sticky header CSS */
.sticky-header {
    position: sticky;
    top: 0; /* Sticks to top when scrolled past */
    z-index: 100; /* Stays above other content */
}

Section 3: Sticky vs Fixed

📌 position: sticky

Starts as relative. Only becomes fixed when you scroll past it. Lives inside its parent container.

📌 position: fixed

Always stays in the same place relative to the viewport. Ignores parent containers.

Section 4: Why Use Sticky?

Sticky headers are perfect for:

  • Navigation bars that stay visible while scrolling
  • Table headers that stay visible while scrolling through large tables
  • Category headers in e-commerce product lists
  • Table of contents sidebars
  • Contact forms that stay visible

🔄 Fixed vs Sticky: Key Differences

position: fixed
  • Always stuck — from the moment the page loads
  • Removed from document flow
  • Positioned relative to viewport
  • Ignores parent containers
  • Example: Chat widget, back-to-top button
position: sticky
  • Starts normal, only sticks when scrolled past
  • Stays in document flow until threshold
  • Positioned relative to parent container
  • Respects parent boundaries
  • Example: Section headers, navigation bars, table headers

📚 Sticky Reference Card

/* Sticky Header */
.sticky-header {
    position: sticky;
    top: 0;
    z-index: 100;
}

/* Sticky Sidebar */
.sticky-sidebar {
    position: sticky;
    top: 20px; /* Sticks 20px from top */
}

/* Sticky Table Header (inside scrollable container) */
.table-container {
    max-height: 400px;
    overflow-y: auto;
}
th {
    position: sticky;
    top: 0;
    background: #1a1a2e;
}

/* 💡 Sticky needs a threshold (top, bottom, left, or right) */
/* 💡 Sticky only works within its parent container */
/* 💡 Parent cannot have overflow: hidden */
